# 智能体状态机信息存储和持久化设计
## 使用组件
- python (编码)
- langgraph (状态机)
- redis (临时存储状态机信息,)
- minio (持久化状态机信息内容)
## 存储结构设计
### REDIS KEY 设计
- 使用 zset + hash 的方式存储状态机内容  
- 状态机快照记录：agent:{userId}:{sessionId}:check_point_list,  ZSET 数据类型存储存放每次快照的id 
- 状态机具体内容: agent:{userId}:{sessionId}:{checkId}, HASH 数据类型，存储每次point的状态机里的内容。
- 过期时间暂时设计10天,防止Redis撑爆
### MINIO 持久化文件存储设计
- 文件分层结构 
```
 文件路径: {project}/{env}/agent/{userId}/session
 文件格式: {sessionId}.jsonl 
 文件内容: 文件内容存储整个REDIS的 zset+hash 格式 [{状态1内容,rediskey},{状态2内,reidiskey}]
```
