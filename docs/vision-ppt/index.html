Background

为什么需要 FusionMark？

📄 文档信息过载

企业每天处理大量 PDF 报告、合同、论文，人工阅读标注效率极低，关键信息容易被遗漏。

🔍 提取精度不足

传统 OCR 和规则提取难以处理复杂排版、表格、公式等多模态元素，准确率难以保证。

🎨 可视化呈现困难

提取结果往往以结构化数据呈现，无法直观地在原文档上标注，不利于人工复核与展示。

⚙️ 配置灵活性差

不同业务场景需要提取不同类别的实体，缺乏可配置、可扩展的提取方案。

What is MinerU

MinerU 是什么？

MinerU 是上海人工智能实验室（OpenDataLab）开源的智能数据提取工具，专注于将复杂多模态 PDF 文档精准转化为清晰、易于分析的 Markdown / JSON 格式。

🏆 开源成绩

GitHub Star 累计飙升 3000+，登顶 GitHub Python Trending，获国内外多个技术大V点赞。

🔬 研发背景

基于 InternVL 多模态大模型技术栈构建，在 WAIC 2024 科学前沿主论坛正式发布。

核心定位

将混合了图片、公式、表格、脚注等在内的复杂多模态 PDF 文档转化为 LLM-ready 的 Markdown / JSON，为 RAG、知识引擎等下游应用提供高质量语料。

FusionMark

Key Features

MinerU 核心功能

📄 多类型 PDF 支持

支持文本型 PDF、图层型 PDF、扫描版 PDF，自动识别文档类型并选择最佳解析策略。

📐 复杂元素解析

精准识别标题、正文、图片、表格、脚注、边注、页眉页脚，自动去除无用元素。

∑ 公式识别与转换

检测行内公式与行间公式区域，使用 UniMERNet 转换为 LaTeX 格式。

🌐 跨平台部署

支持 Windows、macOS、Linux，可选择纯 CPU 或 CUDA / MPS GPU 加速。

PDF-Extract-Kit

MinerU 模型解析工具链

PDF-Extract-Kit 是 MinerU 的核心模型链，由四个关键模块组成，在多个维度性能远超其他开源模型。

1

布局检测 LayoutLMv3-SFT

在 LayoutLMv3-base-chinese 预训练权重基础上做 SFT 微调，检测图像、表格、标题、文本等区域。

2

公式检测 YOLOv8-Trained

基于 YOLOv8l 训练的自研公式检测模型，支持行内公式与行间公式检测。

3

公式识别 UniMERNet

直接使用 UniMERNet 权重，无需额外 SFT，精度在 GitHub 页面有详细验证结果。

4

OCR 识别引擎

高质量 OCR 技术提取准确文本内容，对扫描模糊、水印等情况有较高鲁棒性。

What is LangExtract

LangExtract 是什么？

LangExtract 是 Google 官方推出的轻量级信息提取库，为各种 LLM（如 Google Gemini）提供统一的文本结构化抽取接口，无需微调即可实现高精度实体提取。

🔬 起源

最初应用于医学信息提取场景，被证明在临床文本处理中效果显著，后扩展为通用文本提取库。

🚀 官方认可

Google Developers Blog 官方推荐，作为 Gemini-powered 信息提取方案进行推广。

核心理念

围绕 "Schema 驱动抽取" 思想构建，通过 Prompt 和 Few-shot 示例驱动 LLM 进行精准、可扩展且可验证的信息抽取。

FusionMark

Key Features

LangExtract 核心特性

🧩 Schema 驱动抽取

通过清晰的提取模式定义要抽取的实体类别，LLM 按 Schema 输出结构化结果。

🎓 Few-shot 学习

提供示例数据（ExampleData）展示期望的输出格式，无需微调模型即可获得高精度。

📍 精确来源映射

每个提取的实体都映射到原文的精确字符位置，支持在源文本中高亮显示。

⚡ 并行与优化

支持多 Worker 并行处理、分块优化、多轮提取（extraction_passes）提高召回率。

FusionMark

Schema-Driven

Schema 驱动抽取

LangExtract 使用 ExampleData 和 Extraction 类定义提取模式，LLM 按照定义的结构化格式输出结果。

Step 1

定义提取类别（如 company_name、numeric_value、date）

Step 2

提供 Few-shot 示例展示输入文本与期望提取结果

Step 3

调用 LLM 抽取，输出 AnnotatedDocument 结构化结果

# 定义提取示例
example = data.ExampleData(
    text="苹果出货量 240.6 百万部",
    extractions=[
        data.Extraction(
            class="company_name",
            text="苹果"
        ),
        data.Extraction(
            class="numeric_value",
            text="240.6"
        ),
    ]
)
FusionMark

Few-Shot Learning

Few-Shot 示例学习

LangExtract 的核心是基于示例的学习。你只需提供几个高质量的示例，LLM 就能理解提取模式并泛化到新文本。

💊 医学提取

提取药物名称、剂量、给药途径、频率等医疗实体。

medication · dosage · route · frequency

📈 金融提取

提取公司名称、数值数据、日期、增长率等财经实体。

company_name · numeric_value · date · growth

📚 文学分析

提取人物、情感、情节元素、主题等文学实体。

character · emotion · plot · theme

💡 对于支持结构化输出的提供商（如 Google Gemini），LangExtract 还提供 GeminiSchema 进行字段级 JSON Schema 验证。

Rendering

Markdown → 高亮 PDF 渲染流程

1

LangExtract 实体提取

使用配置中的 extraction_prompt 和 few-shot 示例，调用 LLM 提取结构化实体，输出 List[Extraction(class, text)]。

2

HTML 高亮标记

Markdown → HTML（使用 markdown 库），在文本节点中查找实体，添加 <mark class="highlight-{category}"> 标签，处理重叠匹配（优先保留长匹配）。

3

WeasyPrint 渲染

HTML + CSS → PDF，使用 @page 规则设置页眉页脚，支持中文（Microsoft YaHei / SimSun），输出最终高亮 PDF。
